# 天气查询技能 (get_weather)

获取指定城市的当前天气信息的技能。

## 功能描述

此技能用于获取指定城市的当前天气信息，包括温度、天气状况、风力、湿度、体感温度、能见度、气压等。当用户询问"北京天气怎么样？"、"上海今天温度多少？"、"广州下雨吗？"等问题时，AI 可以调用此技能获取准确的天气信息。

## 使用场景

- 查询城市天气："北京天气怎么样？"
- 查询温度："上海今天多少度？"
- 查询天气状况："广州下雨吗？"
- 查询风力："深圳风大吗？"
- 查询湿度："南京湿度多少？"
- 综合天气查询："告诉我杭州的天气情况"

## 参数说明

### city (必需)
- **类型**: string
- **描述**: 城市名称，支持中文城市名，例如：北京、上海、广州、深圳、杭州、南京等
- **示例**: "北京", "上海", "广州市", "深圳市"

## 返回值

返回 JSON 对象，包含以下字段：

```json
{
  "success": true,
  "city": "北京市",
  "province": "北京市",
  "temp": "15",
  "text": "晴",
  "windDir": "西北风",
  "windScale": "3",
  "humidity": "45",
  "feelsLike": "14",
  "vis": "25",
  "pressure": "1020",
  "updateTime": "2025-03-10T10:30+08:00",
  "description": "北京市当前天气：晴，气温15℃（舒适），微风3级，湿度45%（舒适）。"
}
```

### 字段说明

- **success**: 查询是否成功
- **city**: 城市名称
- **province**: 省份/行政区
- **temp**: 温度（摄氏度）
- **text**: 天气状况文字描述（晴、多云、雨等）
- **windDir**: 风向
- **windScale**: 风力等级
- **humidity**: 湿度（百分比）
- **feelsLike**: 体感温度（摄氏度）
- **vis**: 能见度（公里）
- **pressure**: 气压（百帕）
- **updateTime**: 数据更新时间
- **description**: 人性化的天气描述（AI 可直接使用）

## 技术实现

- **数据源**: 和风天气 API (https://dev.qweather.com/)
- **API 调用**: 使用城市查询 API 获取 locationId，然后调用天气实况 API
- **主脚本**: `skill.py` - 技能核心实现
- **依赖**: `requests` 库（已包含在项目依赖中）
- **错误处理**: 网络超时、城市未找到、API 错误等情况都有相应处理
- **人性化描述**: 自动根据温度、风力、湿度生成人性化的天气描述

### API 使用说明

1. **城市查询**: `https://geoapi.qweather.com/v2/city/lookup`
   - 根据城市名称获取 locationId
   - 返回城市信息和行政区划

2. **天气实况**: `https://devapi.qweather.com/v7/weather/now`
   - 根据 locationId 获取当前天气数据
   - 返回温度、天气状况、风力、湿度等详细信息

## 错误处理

如果执行过程中出现错误，返回格式如下：

```json
{
  "error": "错误描述信息"
}
```

### 常见错误

- **城市未找到**: 当输入的城市名称不正确或不存在时
  ```json
  {
    "error": "城市 '某某' 未找到，请检查城市名称是否正确"
  }
  ```

- **网络请求失败**: 当网络连接出现问题或 API 服务不可用时
  ```json
  {
    "error": "网络请求失败: [具体错误信息]"
  }
  ```

- **请求超时**: 当网络请求超时时
  ```json
  {
    "error": "请求超时，请检查网络连接"
  }
  ```

- **天气数据获取失败**: 当天气 API 返回错误时
  ```json
  {
    "error": "天气数据获取失败，请稍后重试"
  }
  ```
